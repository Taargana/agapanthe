## Intake (passage engine-architect 2026-07-12 acté — pas de re-brainstorm)

- **Problème / nœud** : le hot reload n'a **aucun point d'accroche**. Les 4 pipelines (mesh, shadow, tonemap, skybox) sont construits inline dans le ctor du Renderer (God-object, 848 l.) avec des `GraphicsPipelineDesc` locaux jetés. Pour recréer un pipeline à chaud il faut **capturer** son desc dans un objet vivant → seam par-passe requis d'abord.
- **Dette Graphics découverte (non prévue dans AVANCEMENT)** : `GraphicsPipeline.Dispose()` détruit le handle **immédiatement**, pas via la DeletionQueue. Le spec exige « ancien pipeline → DeletionQueue » (N+2). Chantier à part (M8-02), mirror du patron `GpuImage`.
- **Découpage retenu** : abstraction *légère* (pas de frame graph). Objets-passe dans `Passes/` possèdent le **volatil** (shaders + pipeline + desc-template sans modules + liste des fichiers source résolus) ; le Renderer reste orchestrateur, garde le **stable** (targets, UBOs, samplers, set-layouts, IblResources) et **garde les méthodes `Record*`** (trop couplées à l'état par-frame pour migrer — sur-refactor évité en fin de phase 1). Gate de non-régression : captures **byte-identiques**.
- **Hot reload** : résolveur `#include` maison (retourne la liste des fichiers touchés, détecte cycles + includes manquants = throw §4) ; clé de cache disque = hash du **source résolu après expansion** (corrige un bug latent : aujourd'hui éditer un include n'invalide pas le cache) ; `FileSystemWatcher` sur thread background → **jamais d'appel Vulkan**, drapeau `volatile` + set de chemins sous lock, debounce 150-250 ms, abonnement Changed+Created+Renamed (save atomique temp+rename des éditeurs) ; recréation dans `Renderer.PollShaderReload()` appelé **avant `DrawFrame`** (hors recording), ancien pipeline → DeletionQueue ; échec compile = `Log.Error` + ancien pipeline conservé, pas de crash.
- **Contrainte critique de sûreté** : la deletion différée des pipelines n'est sûre **que parce que** le reload se fait au bord de frame avant recording (N+2 couvre les frames in-flight). Verrouiller par convention/commentaire.

## Project Conventions

- .NET 10, xUnit, TreatWarningsAsErrors, **aucun type Vk* hors Graphics** (PipelineLayoutBuilder `internal` → ne fuit pas), IDisposable partout, destruction différée N+2 (DeletionQueue non-capturante), zéro alloc managée par frame sur le hot path (le reload est HORS hot path — allocs OK ; mais `PollShaderReload` early-out zéro-alloc sur `volatile bool`), ResourceTracker (leak = échec).
